Export in Catima is available int these formats  : **JSON**, **SQL** ou **CSV**.

Depending on your needs, you may chose one or the other:
- **Json or Sql** - usually relevant to share data with a developper. The choice between SQL format or JSON scheme depends on your target database (if you intend to re-create the database outside Catima) or is a matter of personal preference.
- **Csv** - to share data with users in Excel for instance. The CSV export is also relevant if you plan to reimport new data by batch in Catima.


### To summarize

> See below typical use cases 

|  | Catima (JSON)                       | SQL          | CSV          | 
|--------------| ------------------------------------|--------------|--------------|
| **Use cases** | Share data with a developper, Re-create the database | Share data with a developper, Re-create the database | Share with users, Import new data in Catima |
| **Applications**|  PostgreSQL, etc ... |  MYSQL, MariaDB, etc ... | Excel, Numbers | 
| **Useful for import?**| Yes, of *choices*   | No  | Yes, as templates to import data from CSV |
